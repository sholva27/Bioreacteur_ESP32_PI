## 2026-06-21 - Asynchronous Feedback Pattern in Embedded Dashboards
**Learning:** For complex asynchronous UI updates, the \`provideBtnFeedback\` helper returns a 'resolve' function that must be called inside the fetch \`.then()\` callback to transition the button from the interim state (e.g., 'Saving...') to the success state (e.g., 'Saved!'). Calling the helper nestedly (e.g., in a function that calls another function that also uses the helper) can cause the original text to be captured incorrectly as 'Saving...', leading to inconsistent UI states.
**Action:** When nesting asynchronous operations that both provide UI feedback, pass the \`btnId\` as \`null\` to the inner function to suppress redundant feedback loops and prevent 'Saving...' from being captured as the original state.
