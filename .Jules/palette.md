## 2025-05-14 - [Asynchronous Feedback and Error Propagation]
**Learning:** When using asynchronous feedback patterns (like 'Saving...' -> 'Saved!'), it's critical to ensure that errors are correctly propagated through the promise chain. Swallowing an error in a mid-level function (like `updateSettings`) can lead to high-level functions (like `calibrateODZero`) providing false success feedback to the user. Additionally, feedback mechanisms should be targeted to the specific button triggering the action to avoid confusing "leaked" states across the UI.
**Action:** Always re-throw errors in catch blocks within shared utility functions if the caller needs to synchronize UI state with the outcome. Pass explicit identifiers to generic update functions to control which UI elements receive feedback.
